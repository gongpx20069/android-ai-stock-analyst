## 1. TL;DR(一句话结论)

⚠️ **核心矛盾被实测确认了**:你的估值纪律第一指标是「**分析师目标价 → 上行空间%**」,而 **2026 年几乎所有免费行情 API 的"分析师目标价"都已转为付费**(Finnhub 也是)。**唯一能免费、完整拿到目标价+评级+前瞻PE 的来源是 Yahoo Finance(yfinance)**——但它是 Python 库,**安卓端不能直接跑**。

→ 这把决策推向一个**关键二选一**(见 §6):
- **方案①(推荐):** 自建一个极薄后端(FastAPI 包一层 yfinance),App 调它 → 能拿全字段,符合你的框架。
- **方案②(纯无服务器):** App 直连 Finnhub 免费档 → 拿不到目标价,**算不出上行空间%**,你的核心指标缺失。

## 3. 关键发现:免费档"分析师目标价"几乎全军覆没

这是本次调研最重要的一点,直接戳中你的框架:

- **Finnhub**:`Price Target` 和 `Upgrade/Downgrade` 端点在文档侧栏明确标 **Premium(付费)**。免费只剩 `Recommendation Trends`(给出 strongBuy/buy/hold/sell/strongSell 的**分析师家数**,不是目标价)。
- **FMP**:`Price Target Consensus` 端点存在,但免费档(250/天、仅EOD、"Profile and Reference Data")是否含它**不确定**,需拿 key 实测;FMP 近年持续把免费端点上收。
- **其它**(Alpha Vantage/Twelve Data/Alpaca/Polygon):要么没有目标价,要么质量差/受限。

✅ **唯独 yfinance 免费给全**:`targetMedianPrice`(中位目标价)、`numberOfAnalystOpinions`(分析师数)、`recommendationKey`(评级)、`forwardPE`(前瞻PE)一应俱全。

## 3.5 国内源实测:腾讯 / 新浪(2026-06-30 真跑)

你提到腾讯/搜狐也有 API——确实有,我逐个实测了。**它们的最大价值是"国内直连稳、免 key、免翻墙",但拿不到分析师目标价/评级/前瞻PE。**

### ✅ 腾讯(`qt.gtimg.cn`)— 最干净
- 接口:`https://qt.gtimg.cn/q=usNVDA`(美股前缀 `us`,**不带 `.OQ` 后缀**,带后缀会返回 `v_pv_none_match`)
- 实测返回(`~` 分隔):
  ```
  v_usNVDA="200~英伟达~NVDA.OQ~198.69(现价)~194.97(昨收)~197.24(开盘)~63977432(成交量)~...
            ~30.43(TTM PE)~...~236.26(52周高)~151.26(52周低)~...~市值48125亿~..."
  ```
- 能拿:**现价、涨跌、开/昨收、成交量、TTM PE、市值、52周高低**。中文名直接给(英伟达/苹果)。

### ✅ 新浪(`hq.sinajs.cn`)— 需小技巧
- 接口:`https://hq.sinajs.cn/list=gb_aapl`(美股前缀 `gb_` + 小写代码)
- ⚠️ 必须带 `Referer: https://finance.sina.com.cn`,否则被拦;返回是 **GBK 编码**,要转 UTF-8
- 实测:`苹果, 288.38(现价), 2.36(涨%), 时间, ..., 34.74(TTM PE), 317.40(52周高)/200.45(52周低), ...`
- 能拿字段和腾讯类似(现价/涨跌/PE-TTM/市值/52周高低/盘前盘后)。

### ❌ 共同短板(实测确认)
- **没有分析师目标价**、**没有评级**、**没有前瞻PE**(只有 TTM PE)。
- 我连它们 App 端的机构评级逆向接口都试了(`proxy.finance.qq.com/...rank`、`web.ifzq.gtimg.cn/.../UsForecast`、新浪 `US_AnalystRatingService`)——**全部已失效/下线**。
- **搜狐**:只剩 A 股历史行情页面(`q.stock.sohu.com`),无干净美股 API,排除。

### 💡 定位结论
腾讯/新浪 = **优秀的"价格层"**(实时、国内稳、免key),但**补不上你的核心指标(上行空间%)**。→ 催生下面的混合**方案③**。

用 `yfinance` 实跑你持有的三只,字段全部拿到,完全够支撑你的估值纪律:

## 5. GitHub 生态参考(别人怎么解决的)

- **go-stock(⭐6600+,最活跃同类)**:桌面端,自带数据抓取层,AI 用 BYOK。证明"软件免费 + 自己 key"模式成立,但它**不在手机端取数**,绕开了本难题。
- **orthogonal.info 实测博主**的最终方案:**Finnhub(实时价)+ Polygon(EOD)+ yfinance(每周拉一次基本面)三者混用**——侧面印证"没有单一免费源能全包",尤其基本面/分析师数据要靠 yfinance。
- 多数 Flutter 开源股票 App:只做"价格+K线"(用 Finnhub/Twelve Data 直连),**普遍不做分析师目标价**——正因为免费档拿不到。

## 6. 三个现实方案对比(决策 B,登记在 architecture.md)

> 结论:**首选③,其次①,②不推荐**。决策 B 的最终勾选见 [`architecture.md` §3](architecture.md#3-决策台账全项目唯一决策出处);本节只做方案对比论证。

### 方案①:极薄自建后端(FastAPI + yfinance)— ⭐ 推荐
```
[安卓App/Flutter] --HTTPS--> [你的极薄后端 FastAPI] --> yfinance --> Yahoo
                                  ↑ 复用你每天监控那套代码
```
- ✅ **能拿全字段**(目标价/评级/前瞻PE/均线),完全符合你的估值纪律
- ✅ 复用你现成的 yfinance 监控代码,几乎零新开发
- ✅ 后端可缓存,降低 Yahoo 失效影响;App 端逻辑简单
- ❌ 需要一台常开机器(你这台 Ubuntu 即可);yfinance 偶尔需跟进 Yahoo 改版
- 💡 符合你"部署后直接观察、不搞监控告警"的习惯

### 方案②:纯无服务器,App 直连 Finnhub 免费档
```
[安卓App/Flutter] --HTTPS--> Finnhub(现价/评级家数/基本面) 直连
```
- ✅ 零服务器、纯手机、装上即用
- ✅ Finnhub 免费档够慷慨(60次/分)
- ❌ **拿不到分析师目标价 → 算不出"上行空间%"**(你的第一指标缺失)
- ❌ 评级只有"买/持/卖家数",没有中位目标价
- ⚠️ 可用 FMP 免费档补目标价,但需实测是否被锁,且更脆

### 方案③:混合 — App 直连腾讯/新浪取价 + 薄后端 yfinance 取基本面 — ⭐⭐ 最优
```
[安卓App/Flutter] ──直连──> 腾讯/新浪   (实时价/涨跌/PE-TTM,国内秒开免key)
       │
       └──HTTPS──> [薄后端 FastAPI + yfinance]  (目标价/评级/前瞻PE,变化慢、每日缓存)
```
- ✅ **各取所长**:价格层用国内源(快、稳、免翻墙、免key);基本面/分析师层用 yfinance(只有它免费给全)
- ✅ 上行空间%、前瞻PE 全都有 → 完全符合你的估值纪律
- ✅ 后端只需"每天拉一次"分析师数据并缓存,**调用量极小、yfinance 失效影响也小**
- ✅ 实时价走国内源,**用户在中国打开 App 体验最好**(不依赖后端能否访问 Yahoo)
- ❌ 仍需一台常开机器跑薄后端(你这台 Ubuntu 即可);两个数据源需做字段对齐
- 💡 本质是 orthogonal 博主"价格源 + 基本面源分离"实战方案的国内优化版

### 📊 一句话对比
| | ① 薄后端+yfinance | ② 纯直连Finnhub | ③ 国内价+yfinance基本面 |
|---|---|---|---|
| 上行空间%(你的核心) | ✅ 有 | ❌ 没有 | ✅ 有 |
| 前瞻PE | ✅ 有 | ⚠️ Basic Financials | ✅ 有 |
| 实时价国内体验 | ⚠️ 看后端能否连Yahoo | ⚠️ 一般 | ✅ **最好(国内直连)** |
| 要不要养服务器 | 要 | 不要 | 要(但调用量更小) |
| 开发量 | 小 | 中 | 中(两源对齐) |
| 数据稳定性 | 中(Yahoo会改版) | 高 | ✅ 高(价格层不依赖Yahoo) |

## 7. 我的建议

🎯 **首选方案③(国内价 + yfinance 基本面),其次方案①。**

理由:你这个 App 的灵魂是"按我的估值纪律分析",纪律离不开**分析师目标价/上行空间%**(只有 yfinance 免费给全);同时你人在国内、要手机随时看,**实时价用腾讯/新浪直连体验最好、且不依赖后端能否翻墙连 Yahoo**。方案③把"易变的价格"和"慢变的分析师数据"分层,既稳又省——分析师数据每天后台拉一次缓存即可,调用量极小。

> 若想再简化:退回方案①(后端一把抓 yfinance)也行,代价是实时价也走后端、国内访问 Yahoo 稳定性要验证。方案②因缺目标价,**不推荐**。
