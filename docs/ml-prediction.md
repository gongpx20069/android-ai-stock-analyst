# 机器学习股价预测方案

> 这份文档回答:App 里「ML 预测股价」这个**可选**功能怎么实现 + 怎么可视化。
> ⚠️ **先泼一盆冷水(必须诚实)**:股价预测在学术和业界都是出了名的不可靠,市场接近随机游走,任何模型都极易"回测漂亮、实盘拉胯"。所以本功能定位为 **一个辅助参考信号,绝不是买卖依据**——你的估值纪律(上行空间%+前瞻PE)永远是主角,ML 只是配菜。

---

## 0. 设计原则(给 ML 套上缰绳)

1. **辅助而非主导**:ML 预测只作为"额外一栏参考",不参与选股打分主轴,不喂给用户"会涨"的结论。
2. **预测方向 > 预测点位**:预测"未来N天涨/跌概率"比预测"精确到几块几"更现实、更有用。
3. **永远带不确定性**:任何预测都必须配**置信区间/概率**,不给单一确定数字,防止你被一个假精确的数字误导。
4. **可解释优先**:优先用能给出"为什么"的模型(特征重要性),而不是黑箱。
5. **跑在后端**:训练/推理都在你那台 Ubuntu 后端(Python 生态),App 只取结果——手机不适合训练。

---

## 1. 三种方法分层(由轻到重,建议循序渐进)

| 层级 | 方法 | 预测目标 | 实现难度 | 可解释性 | 适合 |
|------|------|---------|---------|---------|------|
| **T1 时间序列** ⭐起步 | Prophet / ARIMA | 未来N天价格区间(带置信带) | ★ 低 | ★★★ 高 | MVP,出"预测带"最快 |
| **T2 树模型** ⭐主力 | XGBoost / LightGBM | 未来N天**涨跌方向+概率** | ★★ 中 | ★★★ 高(特征重要性) | 最佳性价比 |
| **T3 深度学习** | LSTM / GRU / Transformer(TFT) | 价格序列/方向 | ★★★ 高 | ★ 低(黑箱) | 二期可选,收益边际 |

> 业界共识:**对散户自用,T2(LightGBM 做方向分类)性价比最高**——比深度学习更稳、可解释、训练快;T3 深度学习在小数据+高噪声的股价上往往**不比 T2 强多少**,却难调难懂。

---

## 2. 各方法实现细节

### T1:时间序列(Prophet / ARIMA)—— MVP 起步
- **输入**:单只股票历史收盘价(yfinance 拉 1-2 年日线)。
- **库**:`prophet`(Meta 开源,自动处理趋势/季节性,出图自带置信带)或 `statsmodels` 的 ARIMA。
- **输出**:未来 5–30 天的预测曲线 + 80%/95% 置信带。
- **优点**:几行代码出"预测扇形带",直观、可解释。
- **局限**:本质是趋势外推,**不懂基本面/突发事件**,只适合"无大新闻时的平滑预期"。

```python
# 伪代码
from prophet import Prophet
df = yf_history(symbol, period="2y")[["ds","y"]]  # ds=日期, y=收盘价
m = Prophet(interval_width=0.8).fit(df)
forecast = m.predict(m.make_future_dataframe(periods=20))
# forecast: yhat(预测), yhat_lower/upper(置信带)
```

### T2:树模型(LightGBM 方向分类)—— 推荐主力
- **目标**:把"预测价格"改成**分类问题**:未来N天(如5天)收盘是涨还是跌 → 输出**上涨概率%**。
- **选 LightGBM(非 XGBoost)**:两者精度几乎持平(差距被股价噪声淹没),但 LightGBM **训练快 2–10 倍、更省内存、原生支持类别特征**(评级可直接喂),更适合"后端每天重训的自用 App"。样本太小时 XGBoost 过拟合略轻,靠多股票拼训+调 `min_child_samples` 即可抹平。→ 已定,别再纠结。
- **特征工程**(关键,决定上限):
  - 技术类:收益率(1/5/20日)、MA偏离度、RSI、MACD、布林带位置、成交量变化
  - 估值类(你的纪律!):**上行空间%、前瞻PE、评级**(把你的框架变成模型特征)
  - 支撑压力类:**距最近支撑%、距最近压力%**(来自 indicators.md §2)
  - 波动类:历史波动率、ATR
- **库**:`lightgbm`(主力) + `scikit-learn`(切分/评估)。
- **输出**:`上涨概率 0.62`、`特征重要性排名`。
- **关键纪律**:**严格时间切分**(用过去训练、未来测试,绝不能未来数据泄露)+ **滚动回测**。

```python
# LightGBM 方向分类(伪代码)
import lightgbm as lgb
# X: 特征(含上行空间%/RSI/距支撑%...), y: 未来5天涨=1/跌=0
model = lgb.LGBMClassifier(
    num_leaves=31, min_child_samples=30,  # 防过拟合
    n_estimators=200, learning_rate=0.05,
)
model.fit(X_train, y_train)  # 严格时间切分
prob_up = model.predict_proba(X_latest)[:,1]  # → 上涨概率 0.62
importance = model.feature_importances_       # → 可解释
```

### T3:深度学习(LSTM/Transformer)—— 二期可选
- **目标**:用序列模型学价格的时间依赖。
- **库**:`pytorch` + 轻量;或导出 ONNX/TFLite 若想端上推理。
- **现实评估**:对你这种"单股、日线、样本少"的场景,**LSTM 很容易过拟合,实盘提升有限**;Transformer/TFT 更重。
- **建议**:**MVP 不做**,等 T1/T2 跑通且你确实想深入再说。别一上来就上深度学习——投入产出比差。

## 3. ML 结果的可视化(重点,关联 visualization.md)

每种方法都要配一张"诚实展示不确定性"的图:

### 📈 ① 预测扇形带图(Forecast Fan)—— T1 主图
- 形式:历史实线 +未来预测虚线 + **半透明置信带**(越远越宽)。
- 颜色:置信带用浅灰/浅蓝,**强调"区间"而非"一条线"**,防止你盯着单点。
- 作用:一眼看出"模型预期大致方向 + 有多不确定"。

```
价格
 |          实际──────╮
 |                    ╰┄┄┄ 预测中线
 |                  ▒▒▒▒▒▒▒ 95%置信带(越远越宽)
 └─────────────────────────► 时间
        过去        | 未来20天
```

### 📊 ② 涨跌概率仪表 —— T2 主图
- 形式:半圆仪表盘或水平条,指针指向"未来5天上涨概率 62%"。
- 配色:>60%偏绿、40-60%灰(中性)、<40%偏红。
- **必配免责**:旁边小字"模型参考,非买卖建议"。

### 📊 ③ 特征重要性条形图 —— T2 可解释性
- 形式:横向条形,展示"本次预测哪些因素影响最大"(如 上行空间% 0.31、RSI 0.18…)。
- 作用:让预测**可解释**,你能看到模型是不是在用你认可的逻辑(估值)而非乱学噪声。

### 📉 ④ 回测对比图 —— 诚实度检验(最重要!)
- 形式:历史区间内,**预测 vs 实际**两条线叠加 + 标注准确率(方向准确率%、MAE/RMSE)。
- 作用:**让你看清模型到底准不准**,而不是只看一个漂亮的未来预测。**这张图是防止自欺的关键。**

## 4. 架构落点(跑在哪)

```
[你的 Ubuntu 后端]
  ├─ 拉数据(yfinance 历史日线)
  ├─ 训练/加载模型(LightGBM/Prophet)── 每日/每周离线训练一次,存模型文件
  ├─ 推理 → 输出 {预测带, 涨跌概率, 特征重要性, 回测指标}
  └─ 暴露 GET /predict/{symbol} ── 返回 JSON
        │
        ▼
[Flutter App] 取 JSON → 用 fl_chart 画 ①②③④ 图
```
- **训练在后端离线做**(慢、重),App 只取**已算好的结果**(秒回)。
- 与 data-sources.md 方案①/③天然契合——反正都要后端,ML 顺手挂上去。
- 模型按天/周重训即可(股价特征变化没那么快),不需要实时训练。

---

## 5. ⚠️ 风险与红线(写进产品)

1. **必带免责声明**:每个预测页固定显示"模型预测仅供参考,非投资建议,历史不代表未来"。
2. **不显示"精确点位预测"做主结论**:主打概率+区间,弱化单点数字。
3. **回测要诚实**:展示真实方向准确率(通常 50-60% 就算不错,别美化)。
4. **防过拟合**:严格时间序列切分、滚动验证、特征不泄露未来。
5. **不与"选股打分"混合**:ML 预测是详情页的**独立一栏**,不污染你基于估值的选股主逻辑。

> 与你的投资框架对齐:你本来就**偏好历史趋势分析、反对价格预测**。所以这个 ML 模块的正确姿势是——**"给个参考概率 + 老实展示它有多不准",而不是"告诉你买它会涨"。**
