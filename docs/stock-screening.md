# 选股技术方案

> 这份文档回答:App 里「选股」功能到底怎么实现。核心思路 —— **把"我现成的选股流水线"+"我的估值纪律"编码进 App,而不是让 AI 瞎推荐。**

---

## 0. 设计原则(必须先立规矩)

1. **数据驱动、非荐股**:输出"按规则筛出的候选 + 客观数据",不做"买它会涨"的预测。
2. **复用现成流水线**:我已有 `us-stock-screening` skill(nasdaq universe → 过滤 → yfinance 基本面 → 分析),选股逻辑直接搬。
3. **估值纪律内嵌**:排序/打分以**上行空间% + 前瞻PE**为主,贯彻"在跌≠便宜,要看上行空间"。
4. **AI 只做最后一层解读**,不参与"选谁",避免幻觉选股。

---

## 1. 选股流水线(四步漏斗)

```
[全市场 universe]  →  [硬性过滤]  →  [估值打分排序]  →  [AI 解读 Top N]
   ~5000+只            ~几百只           Top 20-50          Top 5-10
   nasdaq-public      规则筛(秒级)     按你的纪律打分     喂给你的Azure key
```

### 第1步:Universe(候选池)
- 来源:`nasdaq-public` 的 screener,拉全 Nasdaq/NYSE 代码表(分页取全)。
- 免 key、免后端;数据量大但只是代码+基础字段,很轻。

### 第2步:硬性过滤(快、纯规则,先砍掉一大半)
沿用 `us-stock-screening` 默认门槛,可在 App 里调:
- 市值 ≥ $10B(默认,过滤小盘/老千)
- 现价 ≥ $5(过滤仙股)
- 日均成交量 ≥ 500K(保证流动性)
- 有明确行业/板块(去掉空白)
- (可选)行业白名单/黑名单 —— 比如只看科技、剔除生物医药等高波动

### 第3步:估值打分排序 ⭐(你的纪律就编码在这)
对过滤后的票,用 yfinance 拉关键字段,按**你的框架**算分:

| 维度 | 字段 | 规则 | 权重(建议) |
|------|------|------|-----------|
| **上行空间%** | `targetMedianPrice` ÷ `currentPrice` − 1 | 越大越好;<0 直接淘汰 | 40% |
| **前瞻PE** | `forwardPE` | 越低越好;为负/极高降权 | 30% |
| **分析师置信** | `numberOfAnalystOpinions` | <5 家降权(置信不足) | 10% |
| **评级** | `recommendationKey` | buy/strong_buy 加分,sell 淘汰 | 10% |
| **位置(防追高)** | 现价 vs 52周高 / 200日均线 | 离52周高太近(如>95%)降权 | 10% |

> 关键:**排序主轴是"上行空间% + 低前瞻PE",不是"涨得多"。** 这正是你"避免买在山顶"的核心,把它写成代码而非凭感觉。

输出:一张按总分排序的候选表(代码/名称/现价/中位目标价/上行空间%/前瞻PE/评级/分析师数)。

### 第4步:AI 解读 Top N(用你的 Azure key)
- 只把排名靠前的 5–10 只,连同结构化数据,喂给你自己的 Azure OpenAI。
- AI 的任务是**解读 + 风险提示**(为什么便宜?上行空间是否可信?有何风险?),**不是重新选股**。
- Prompt 内嵌你的纪律(见 `ai-prompt.md`,待写):看上行空间+低PE、在跌≠便宜、三大卖出信号。

---

## 2. 两种"选股入口"(App 里的功能形态)

| 入口 | 场景 | 实现 |
|------|------|------|
| **A. 自选监控**(高频用) | 看我已持仓/关注的票(GOOGL/AMZN/NVDA…) | 直接对固定列表跑第3-4步,**等于把你每日 WeChat 监控搬进 App** |
| **B. 全市场筛选**(低频用) | "帮我从全市场挑几只低估的" | 完整跑第1-4步漏斗 |

> 建议 MVP **先做 A(自选监控)**:复用度最高、数据量小、最快出成果;B(全市场筛选)二期再上。

---

## 3. 性能与额度现实(重要)

全市场筛选最大的坑是**第3步要对几百只逐个调 yfinance**,慢且易触发限流。对策:

1. **两段式**:硬过滤(第2步)在端上/后端用轻字段快速砍到几百只;只对**幸存者**调 yfinance 重字段。
2. **后端缓存 + 批量**:基本面/分析师数据变化慢,**后端每天批量拉一次全市场缓存**,App 查的是缓存 → 秒回、零限流。这与 `data-sources.md` 方案③的"后端每日缓存"天然契合。
3. **分页/懒加载**:候选表先出 Top 20,滑动再加载。

---

## 4. 与数据源方案的衔接

- 选股**重度依赖分析师目标价/前瞻PE** → 再次印证 `data-sources.md` 里**必须有 yfinance 那一层**(方案①或③),方案②(纯Finnhub)做不了这套打分。
- 实时价可走腾讯/新浪(方案③),但**目标价/前瞻PE 这些打分字段只能来自后端 yfinance 缓存**。
