# 架构设计与待定决策

> 本文是讨论主战场。每个"❓决策点"都需要我们一起拍板,定了就更新这里。

---

## 0. 一句话目标

做一个**安卓 App**:输入股票代码 → 拉行情 + 算基础量化指标 → 调用**用户自己的 Azure OpenAI** 做 AI 解读 → 输出一份"按我的估值纪律"写的分析报告。

---

## 1. 关键矛盾(必须先想清楚)

手机端做股票分析,最大难点**不是 AI,而是数据从哪来**。手机 App 直接抓行情会遇到:
- yfinance 是 Python 库,**安卓上不能直接跑**;
- 免费行情 API 大多有调用限制 / 需要 key / 国内访问不稳;
- 不想自己养服务器(养服务器=不再"纯本地免费")。

所以**技术栈 + 数据源是一对绑定决策**,见下面 §2 §3。

---

## 2. ✅ 决策点 A:技术栈 —— 已定:**Flutter / Dart**

**结论:用 Flutter(Dart)写真·安卓 APK。** Java 直接排除(2026 新项目首选 Kotlin,不用 Java)。

定它的核心理由:
1. **复用 mochi-pet 经验** —— 同样用 Flutter,环境/打包 APK/签名/发布整套已趟过。
2. **本 App 重头戏是 UI 展示**(股票卡片、估值表、K线、AI 报告排版),Flutter 控件丰富、`fl_chart` 画 K 线顺手,比原生省事。
3. **网络请求(拉行情 + 调 Azure)两端一样轻松**,原生 Kotlin 的性能/体积优势在这个 App 上几乎用不到。
4. 一套代码以后还能出 iOS。

> 备忘:原本备选 A2 原生 Kotlin / A3 Python 套壳 / A4 网页 PWA,均已放弃。Python 偏好在这个项目让位于"复用现有 Flutter 经验"。

---

## 3. ❓决策点 B:行情数据从哪来

> 📄 **已完成专项调研 → 见 [`data-sources.md`](data-sources.md)**。关键结论:2026 年免费 API 的"分析师目标价"几乎全转付费,**只有 yfinance 免费给全**(实测已验证你持仓三只字段齐全),但它是 Python 库、安卓端不能直接跑。

收敛为两个现实方案(详见调研报告 §6):

| 方案 | 能否拿"上行空间%" | 要养服务器? | 开发量 | 推荐 |
|------|------------------|------------|--------|------|
| **①薄后端(FastAPI+yfinance)** | ✅ 能(核心指标全) | 要(你现有 Ubuntu 即可) | 小(复用监控代码) | ⭐ |
| **②纯直连 Finnhub 免费档** | ❌ 不能(无目标价) | 不要 | 中 | — |

> 建议走**方案①**:你的估值纪律离不开分析师目标价,而这只有 yfinance 免费给全;后端可复用每日监控代码,并可同时接 Finnhub 当价格备份源。**等你拍板。**

## 4. ❓决策点 C:量化指标做哪些(MVP 范围)

先做"够用且符合你框架"的最小集,别贪多:

- ✅ **估值面**(你的核心):现价、中位目标价、上行空间%、前瞻 PE、分析师数量与评级
- ✅ **技术面**(辅助):50/200 日均线、距均线%、RSI、52周高低位置
- ✅ **盈亏**:输入成本 → 自动算浮盈亏%
- ⏸️ 暂不做:期权、财报详表、回测(二期再说)

> ❓你还想加什么?(比如资金流、情绪、板块对比)

---

## 5. ❓决策点 D:AI 接入(Azure OpenAI BYOK)

- 用户在设置里填:**Endpoint / API Key / Deployment Name / API Version**(Azure 四件套,跟普通 OpenAI 不同)。
- key **只存本地**(安卓 EncryptedSharedPreferences / 本地加密),绝不上传。
- AI 的角色:**把结构化数据 + 你的估值纪律喂给模型,让它输出中文解读**,而不是让它瞎预测涨跌。
- Prompt 模板单独写在 `docs/ai-prompt.md`(把"看上行空间+低PE、在跌≠便宜、三大卖出信号"编码进去)。

---

## 6. ❓决策点 E:数据/安全边界

- key、自选股、成本价 → 全部**本地存储**,不建账号、不上云。
- 行情数据缓存本地,减少 API 调用。
- 开源协议:建议 **GPL-3.0 或 MIT**(待定)。
