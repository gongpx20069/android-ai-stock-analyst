# 架构设计与待定决策

> 本文是讨论主战场。每个"❓决策点"都需要我们一起拍板,定了就更新这里。

---

## 0. 一句话目标

做一个**安卓 App**:输入股票代码 → 拉行情 + 算基础量化指标 → 调用**用户自己的 Azure OpenAI** 做 AI 解读 → 输出一份"按我的估值纪律"写的分析报告。

---

## 1. 关键矛盾(必须先想清楚)

手机端做股票分析,最大难点**不是 AI,而是数据从哪来**。手机 App 直接抓行情会遇到:
- yfinance 是 Python 库,**安卓上不能直接跑**;
- 免费行情 API 大多有调用限制 / 需要 key / 国内访问不稳;
- 不想自己养服务器(养服务器=不再"纯本地免费")。

所以**技术栈 + 数据源是一对绑定决策**,见下面 §2 §3。

---

## 2. ❓决策点 A:技术栈怎么选

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **A1. Flutter** | 一套代码出安卓(以后能出 iOS) | 生态成熟、UI 漂亮;你熟悉的 mochi-pet 就是 Flutter | Dart 语言,不是你偏好的 Python |
| **A2. 原生 Kotlin** | 安卓原生 | 性能最好、最"正统" | 学习曲线陡,只服务安卓 |
| **A3. Python 套壳**(Kivy/BeeWare/Flet) | 用 Python 写安卓 | **你偏好 Python** | 打包 APK 麻烦、生态弱、行情库未必能用 |
| **A4. 网页 App(PWA)+ 手机浏览器** | 后端 Python,前端网页,手机当浏览器开 | **复用你现有 yfinance 那套**;开发最快 | 严格说不是"APK",但能"添加到主屏幕"像 App |

> 💡 我的倾向先抛出来:**如果你最看重"能快速跑通 + 复用现有 Python 行情代码"→ A4;如果你执意要真·APK 且愿学一点 → A1 Flutter。** 等你定。

---

## 3. ❓决策点 B:行情数据从哪来

| 方案 | 免费? | 在手机端可行? | 备注 |
|------|-------|---------------|------|
| **B1. 自建轻后端**(FastAPI + yfinance) | 免费(自己跑) | App 调自己后端 | 复用你现有代码;但要有台机器常开 |
| **B2. 免费行情 API**(Alpha Vantage / Finnhub 等) | 有免费额度 | ✅ App 直连 | 需各自注册 key;有频率限制(如 25–60 次/天) |
| **B3. 券商/数据商 API** | 多数收费 | ✅ | 数据好但要钱,排除 |

> ⚠️ 关键权衡:**B1 复用你已有的 yfinance 监控代码、数据最全,但需要一台常开机器(可以就是你这台?);B2 纯手机直连、零服务器,但额度受限、字段没 yfinance 全。**

## 4. ❓决策点 C:量化指标做哪些(MVP 范围)

先做"够用且符合你框架"的最小集,别贪多:

- ✅ **估值面**(你的核心):现价、中位目标价、上行空间%、前瞻 PE、分析师数量与评级
- ✅ **技术面**(辅助):50/200 日均线、距均线%、RSI、52周高低位置
- ✅ **盈亏**:输入成本 → 自动算浮盈亏%
- ⏸️ 暂不做:期权、财报详表、回测(二期再说)

> ❓你还想加什么?(比如资金流、情绪、板块对比)

---

## 5. ❓决策点 D:AI 接入(Azure OpenAI BYOK)

- 用户在设置里填:**Endpoint / API Key / Deployment Name / API Version**(Azure 四件套,跟普通 OpenAI 不同)。
- key **只存本地**(安卓 EncryptedSharedPreferences / 本地加密),绝不上传。
- AI 的角色:**把结构化数据 + 你的估值纪律喂给模型,让它输出中文解读**,而不是让它瞎预测涨跌。
- Prompt 模板单独写在 `docs/ai-prompt.md`(把"看上行空间+低PE、在跌≠便宜、三大卖出信号"编码进去)。

---

## 6. ❓决策点 E:数据/安全边界

- key、自选股、成本价 → 全部**本地存储**,不建账号、不上云。
- 行情数据缓存本地,减少 API 调用。
- 开源协议:建议 **GPL-3.0 或 MIT**(待定)。

---

## 7. 建议的推进顺序(等技术栈定了再细化)

1. 定 §2 技术栈 + §3 数据源(**最优先,其它都依赖它**)
2. 写死一个 demo:命令行/脚本拉 1 只股票的全字段 → 验证数据通路
3. 接 Azure OpenAI,跑通"数据 → AI 报告"
4. 包 UI(App 或网页)
5. 打包 APK / 部署
