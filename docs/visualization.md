# 可视化设计方案

> 这份文档回答:App 里「图表/可视化」怎么做。原则 —— **可视化要服务你的估值纪律(看上行空间、防追高),而不是堆花哨图表。**

---

## 0. 设计原则

1. **每张图都对应一个决策**:不画"好看但没用"的图;每张图要能直接支撑"该不该买/卖/持"。
2. **估值优先于价格**:纪律是看上行空间%+前瞻PE,所以**估值类图是主角,K线是配角**。
3. **一眼可读 + 中文标注**:手机小屏,重点用数字/颜色突出;Flutter 端绘图(非 matplotlib),中文无缺字问题。配色统一走 [MASTER §2/§5](design-system/ai-stock-analyst/MASTER.md) 的反直觉估值语义(绿=有空间、红=追高),不在本文另定色值。

---

## 1. Flutter 端图表库选型

| 库 | 适合 | 取舍 |
|----|------|------|
| **fl_chart** ⭐主用 | 折线/柱状/饼/雷达/散点 | 纯Dart、活跃、零授权顾虑;**估值类图全覆盖**。K线需叠 candlestick 扩展 |
| **syncfusion_flutter_charts** | 专业金融图(自带K线/布林/均线/RSI) | 商业库(社区版个人免费,需注册License);K线二期用 |
| **candlesticks** | 专门K线,开箱即用+缩放 | 只做K线,配合 fl_chart |

> **选型结论**(已在 [`architecture.md` §3](architecture.md#3-决策台账全项目唯一决策出处) 默认采纳):主用 fl_chart 覆盖 P0/P1 估值与盈亏图,K线二期用 candlesticks 或 syncfusion;MVP 可先不做 K线。每张图的配色/A11y 规则见 [MASTER §5](design-system/ai-stock-analyst/MASTER.md),本文只管"画哪些图"。

## 2. 核心可视化清单(按优先级)

### 🥇 P0 —— 估值类(你的纪律核心,MVP 必做)

**① 上行空间条形图(Upside Bar)** —— 最重要
- 形式:横向条形,每只股票一条,长度=上行空间%((中位目标价÷现价−1)),标注现价、中位目标价两端点。
- 作用:**一眼看出哪只"离目标价还有空间",直接服务"在跌≠便宜,要看上行空间"。**

```
NVDA   现价198 ──────────────●目标294   +48% 🟢
GOOGL  现价356 ────────●目标430          +21% 🟢
AMZN   现价238 ─────●目标315             +32% 🟢
```

**② 现价 vs 目标价 区间点图(防追高)**
- 形式:一根从 52周低 → 52周高 的横轴,标出 现价 / 200日均线 / 中位目标价 三个点。
- 作用:看现价在年度区间的**位置**;贴近52周高=追高预警(红),贴近低位+有上行空间=机会(绿)。

**②b 支撑压力位图(你强调必做)** —— 详情页核心
- 形式:以现价为中心的纵向价格轴,把 indicators.md §2 算出的**支撑位(下方)/压力位(上方)**标在轴上,注来源与距现价%,**≥2种方法共振的位置加粗**。配色按 MASTER §5(方向线走中性色,不抢估值红绿)。
- 作用:一眼看"上方到压力还有多少空间、下方跌到哪有支撑",判断风险收益比(图示见 indicators.md §2 综合呈现)。

**③ 前瞻PE 对比柱状图**
- 形式:多只股票的 forwardPE 并排柱状,叠一条"行业/历史均值"参考线。
- 作用:横向比"谁更便宜",贯彻低PE偏好;为负/异常值特殊标记。

### 🥈 P1 —— 持仓/盈亏类(自选监控入口用)

**④ 盈亏概览**
- 形式:每只持仓一行:成本→现价的迷你条 + 浮盈亏%(绿/红)。
- 作用:把你每日 WeChat 监控报告的表格**可视化**;一眼看整体仓位健康度。

**⑤ 评级分布(可选)**
- 形式:小型堆叠条或饼图,展示 strongBuy/buy/hold/sell 分析师家数占比。
- 作用:辅助判断市场共识强弱。

### 🥉 P2 —— 价格/技术类(二期,配角)

**⑥ K线 + 均线(50/200日)**
- 形式:蜡烛图叠 MA50/MA200;标注金叉/死叉。
- 作用:技术面辅助,**但明确是配角**,避免你被短期价格带偏。

**⑦ RSI / 52周位置仪表**
- 形式:RSI 折线(超买70/超卖30参考线)或半圆仪表盘显示52周位置百分位。
- 作用:辅助判断是否过热/超卖。

## 3. 页面布局建议(可视化落到哪)

| 页面 | 放哪些图 |
|------|---------|
| **个股详情页** | ②区间点图 + ⑥K线 + ⑤评级分布 + 关键数字卡片(现价/目标/上行%/前瞻PE/评级) |
| **自选监控页** | ①上行空间条形图(多只横排) + ④盈亏概览 |
| **筛选结果页** | ③前瞻PE对比柱状 + ①上行空间排序条形 |
| **AI 解读页** | 文字报告为主,顶部嵌一张①或②做"数据快照" |

## 4. 数据 → 图 的字段映射

| 图 | 依赖字段 | 来源(对应 data-sources) |
|----|---------|----------------------|
| ①上行空间 | currentPrice, targetMedianPrice | 价格→国内源;目标价→yfinance后端 |
| ②区间点图 | currentPrice, 52周高低, 200日均线 | 价格/52周→国内源或yfinance;均线→yfinance |
| ③前瞻PE | forwardPE | yfinance后端 |
| ④盈亏 | currentPrice + 用户成本(本地存) | 价格→国内源;成本→本地 |
| ⑥K线 | OHLC 历史 | yfinance后端 / 腾讯K线接口 |

> 再次印证:**估值类图(①②③)离不开 yfinance 的目标价/前瞻PE**(与 data-sources §6 方案①/③一致);纯国内源/Finnhub 画不出上行空间图。
